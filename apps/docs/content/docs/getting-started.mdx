## Launch

Start Agent Dash from any terminal. It connects to your running `tmux` server and discovers agent sessions automatically — no config, no registration.

```bash
$ agent-dash
# or, if you prefer it to exit when you attach to a pane:
$ agent-dash --exit
```

If nothing shows up, Agent Dash isn't finding any Claude processes. Start one in any tmux pane and it will appear on the next scan tick.

## The interface, in two panes

Agent Dash is a dual-pane TUI. On the left: every session it's found, grouped by tmux session name. On the right: a live preview of the one you've selected, rendered with full ANSI fidelity.

<ul className="prompt">
  <li><b>Session list</b> — left pane. One row per Claude process. Groups collapse with <span className="ad-kb-inline"><kbd>l</kbd></span>.</li>
  <li><b>Preview pane</b> — right pane. Low-latency FIFO stream with a `capture-pane` fallback.</li>
  <li><b>Status bar</b> — bottom strip. Active/unread counts on the left, pinned keybinds on the right.</li>
</ul>

### Navigating sessions

Movement is vim-shaped. <span className="ad-kb-inline"><kbd>j</kbd></span> and <span className="ad-kb-inline"><kbd>k</kbd></span> step through sessions, arrow keys work if you prefer. <span className="ad-kb-inline"><kbd>0</kbd></span> focuses the preview pane; <span className="ad-kb-inline"><kbd>1</kbd></span> focuses the list.

### Switching panes

When you want to actually type into a session, press <span className="ad-kb-inline"><kbd>o</kbd></span>. Agent Dash will attach your terminal to that tmux pane directly. <span className="ad-kb-inline"><kbd>O</kbd></span> opens it in a popup instead, leaving Agent Dash in place.

| Key | Action |
|-----|--------|
| `j` / `↓` | Next session |
| `k` / `↑` | Previous session |
| `o` | Switch to the tmux pane |
| `O` | Open the pane in a popup |
| `r` | Mark session as read |
| `?` | Toggle the help overlay |

## Session groups

Sessions sharing a tmux session name are grouped together. Use <span className="ad-kb-inline"><kbd>l</kbd></span> to expand a group and <span className="ad-kb-inline"><kbd>h</kbd></span> to collapse it. Press <span className="ad-kb-inline"><kbd>H</kbd></span> to hide/unhide a session or group. If you'd rather see a flat list, the backtick toggles it:

```text
# toggle between grouped and flat view
` — toggle flat view
h — collapse a group
H — hide/unhide a session or group
l — expand a group or the hidden section
```

<Callout kind="note" title="Note · hidden sessions aren't gone">
  Hidden sessions live in a collapsed "`─── Hidden (n) ───`" row at the bottom of the list. Expand it with <span className="ad-kb-inline"><kbd>l</kbd></span> to see or un-hide them.
</Callout>

## Copy mode

Press <span className="ad-kb-inline"><kbd>v</kbd></span> in the preview pane to freeze it and enter copy mode — the same motions you know from vim, plus forward/backward search.

<ul className="prompt">
  <li><b>Movement</b> — `hjkl`, `w b e`, `0 $`, `gg G`.</li>
  <li><b>Selection</b> — <span className="ad-kb-inline"><kbd>v</kbd></span> starts a selection, <span className="ad-kb-inline"><kbd>y</kbd></span> yanks to the system clipboard.</li>
  <li><b>Search</b> — <span className="ad-kb-inline"><kbd>/</kbd></span> forward, <span className="ad-kb-inline"><kbd>?</kbd></span> backward, <span className="ad-kb-inline"><kbd>n</kbd></span> / <span className="ad-kb-inline"><kbd>N</kbd></span> to step through matches.</li>
  <li><b>Exit</b> — <span className="ad-kb-inline"><kbd>Esc</kbd></span> returns to live preview.</li>
</ul>

## Where state lives

Agent Dash persists everything you'd expect: read markers, collapsed groups, hidden sessions, and cached previews. You can inspect or clear any of it by hand.

| Path | Purpose |
|------|---------|
| `~/.config/agent-dash/config.json` | Configuration |
| `~/.config/agent-dash/state.json` | Read markers, visibility, collapse state |
| `~/.config/agent-dash/cache/` | Preview + session cache |
| `/tmp/agent-dash-{pid}-preview.fifo` | Live preview FIFO |

<Callout kind="warn" title="Warning · FIFO cleanup">
  FIFOs are created with `O_NONBLOCK` in `/tmp`. If Agent Dash crashes hard, a stale FIFO may be left behind — it's harmless, and the next launch will replace it.
</Callout>
